Demystifying Softmax Loss: A Step-by-Step Derivation for Linear Classifiers

If you are diving into the mechanics of neural networks, you will inevitably encounter the backpropagation of the Softmax and Cross-Entropy loss. At first glance, the matrix calculus can feel a bit intimidating. However, once you break it down step-by-step using the chain rule, you will discover that the final gradients are incredibly elegant and intuitive.

In this post, we will walk through the complete mathematical derivation of the gradients for a linear classifier, moving from single variables to full matrix vectorization.

1. The Setup: Forward Propagation

Let’s define our variables for a single training sample. Assume our input features have a dimension of DDD, and we are classifying them into CCC distinct classes.

  • Input xxx: A D×1D \times 1D×1 column vector.
  • Weights WWW: A C×DC \times DC×D matrix.
  • Bias bbb: A C×1C \times 1C×1 column vector.
  • True Label yyy: The true class index (or a C×1C \times 1C×1 one-hot encoded vector where only the true class index is 111).

The Linear Layer (Logits):
First, we compute the raw scores (logits) zzz for each class:

z=Wx+bz = Wx + b z=Wx+b

The Softmax Layer:
We convert these raw scores into a valid probability distribution. The probability of the sample belonging to class iii is:

pi=ezi∑j=1Cezjp_i = \frac{e^{z_i}}{\sum_{j=1}^C e^{z_j}} pi​=∑j=1C​ezj​ezi​​

The Cross-Entropy Loss:
For a single sample where the true class is yyy, the loss only cares about the predicted probability assigned to that true class:

L=−log⁡(py)L = -\log(p_y) L=−log(py​)

By substituting the Softmax formula into the loss function, we get:

L=−zy+log⁡(∑j=1Cezj)L = -z_y + \log\left(\sum_{j=1}^C e^{z_j}\right) L=−zy​+log(j=1∑C​ezj​)

2. The Core Derivation: Gradient with respect to Logits (zzz)

To perform backpropagation, we first need to find how the loss changes with respect to each logit ziz_izi​. We denote this gradient as ∂L∂zi\frac{\partial L}{\partial z_i}∂zi​∂L​.

We must split this into two scenarios: when iii is the true class, and when iii is any other class.

Case 1: Deriving for the true class (i=yi = yi=y)

∂L∂zy=∂∂zy(−zy+log⁡(∑j=1Cezj))\frac{\partial L}{\partial z_y} = \frac{\partial}{\partial z_y} \left( -z_y + \log\left(\sum_{j=1}^C e^{z_j}\right) \right) ∂zy​∂L​=∂zy​∂​(−zy​+log(j=1∑C​ezj​))

Applying the chain rule to the logarithm term:

∂L∂zy=−1+1∑j=1Cezj⋅ezy\frac{\partial L}{\partial z_y} = -1 + \frac{1}{\sum_{j=1}^C e^{z_j}} \cdot e^{z_y} ∂zy​∂L​=−1+∑j=1C​ezj​1​⋅ezy​

Notice that the fractional term is exactly our definition of pyp_ypy​!

∂L∂zy=py−1\frac{\partial L}{\partial z_y} = p_y - 1 ∂zy​∂L​=py​−1

Case 2: Deriving for an incorrect class (i≠yi \neq yi=y)
Because zyz_yzy​ does not contain ziz_izi​, the derivative of the first term is 000.

∂L∂zi=0+1∑j=1Cezj⋅ezi\frac{\partial L}{\partial z_i} = 0 + \frac{1}{\sum_{j=1}^C e^{z_j}} \cdot e^{z_i} ∂zi​∂L​=0+∑j=1C​ezj​1​⋅ezi​

Again, the fractional term is pip_ipi​:

∂L∂zi=pi\frac{\partial L}{\partial z_i} = p_i ∂zi​∂L​=pi​

The Vectorized Form:
We can beautifully combine these two cases using an indicator function I(i=y)\mathbb{I}(i=y)I(i=y) (which is 111 if iii is the true class, and 000 otherwise).

Let dzdzdz be the gradient vector ∂L∂z\frac{\partial L}{\partial z}∂z∂L​. Its vectorized form is simply:

dz=p−ydz = p - y dz=p−y

Intuition Check: This result is incredibly logical. The gradient is simply the Predicted Probability minus the True Probability. If the model is 100% confident and correct (py≈1p_y \approx 1py​≈1), the gradient is 000, and no weights will be updated. The larger the error, the larger the gradient pushing the model to learn.

3. Gradients with respect to Weights (WWW) and Bias (bbb)

Now that we have the gradient of the loss with respect to the logits (dzdzdz), we use the chain rule to pass this error signal back to our parameters WWW and bbb.

Deriving for WWW:
Let’s look at a single weight element WijW_{ij}Wij​. It only influences the loss through the specific logit ziz_izi​.

∂L∂Wij=∂L∂zi⋅∂zi∂Wij\frac{\partial L}{\partial W_{ij}} = \frac{\partial L}{\partial z_i} \cdot \frac{\partial z_i}{\partial W_{ij}} ∂Wij​∂L​=∂zi​∂L​⋅∂Wij​∂zi​​

Since zi=∑Wikxk+biz_i = \sum W_{ik} x_k + b_izi​=∑Wik​xk​+bi​, the local gradient ∂zi∂Wij\frac{\partial z_i}{\partial W_{ij}}∂Wij​∂zi​​ is simply xjx_jxj​. Therefore:

∂L∂Wij=dzi⋅xj\frac{\partial L}{\partial W_{ij}} = dz_i \cdot x_j ∂Wij​∂L​=dzi​⋅xj​

To vectorize this back into a C×DC \times DC×D matrix (the same shape as WWW), we take the outer product of the column vector dzdzdz and the row vector xTx^TxT:

∂L∂W=dz⋅xT\frac{\partial L}{\partial W} = dz \cdot x^T ∂W∂L​=dz⋅xT

Deriving for bbb:
Since z=Wx+bz = Wx + bz=Wx+b, the local derivative ∂z∂b\frac{\partial z}{\partial b}∂b∂z​ is 111. Thus, the error signal passes directly through:

∂L∂b=dz\frac{\partial L}{\partial b} = dz ∂b∂L​=dz

4. Scaling up: The Mini-Batch Form

In real-world training, we process NNN samples at a time to stabilize gradients and utilize parallel computing.

  • XXX becomes a D×ND \times ND×N matrix.
  • dZ=P−YdZ = P - YdZ=P−Y becomes a C×NC \times NC×N matrix.

To find the average gradient across the entire batch, we perform a matrix multiplication and divide by NNN:

Batch Weight Gradient:

∂L∂W=1NdZ⋅XT\frac{\partial L}{\partial W} = \frac{1}{N} dZ \cdot X^T ∂W∂L​=N1​dZ⋅XT

Batch Bias Gradient:
Sum the errors across all NNN samples for each class, then average:

∂L∂b=1N∑i=1NdZ(i)\frac{\partial L}{\partial b} = \frac{1}{N} \sum_{i=1}^N dZ^{(i)} ∂b∂L​=N1​i=1∑N​dZ(i)

Summary

By systematically applying the chain rule, we transformed a seemingly complex matrix calculus problem into clean, highly efficient linear algebra operations. Understanding this dz=p−ydz = p - ydz=p−y dynamic is the fundamental key to grasping how classification networks “learn” from their mistakes.
