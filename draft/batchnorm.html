BatchNorm：在漂移的中间，给每一层一个暂时稳定的早晨

有一段时间，我对训练深层网络的印象，很像看天气。

前几个 epoch 还算晴朗，loss 也在往下走；再过一会儿，曲线忽然开始颤，像风把水面吹皱。学习率不敢开大，初始化要反复试，activation 稍微选得不对，梯度就可能在某一层之后变得迟钝，或者干脆失控。你会觉得问题出在很多地方，但又说不清到底是哪里先开始松动。

Batch Normalization，或者更常被叫作 BatchNorm，就是在这样的背景下出现的。它不是那种气势很大的方法。它做的事其实很朴素：当一个 mini-batch 流过某一层时，先把这一层的输出拉回一个比较稳定的尺度，再交给下一层。

问题先在哪里

先把场景讲清楚。

神经网络不是一层一层独立训练的。当前面某一层的参数更新后，后面层收到的输入分布也会跟着变。今天这一层看到的 activation，和几百步之后看到的 activation，均值可能变了，方差也可能变了。对后面的层来说，它像是在一块一直轻微移动的地面上学习站稳。

BatchNorm 论文最初用了一个很有名的说法来描述这件事：internal covariate shift。这个词后来有不少争议，因为后续研究发现，BatchNorm 的效果并不能完全用“中间层输入分布变了”这一句话解释完。但作为直观入口，它仍然很好用：如果每一层收到的数值范围总在漂，优化自然会变得辛苦。

所以 BatchNorm 的第一个直觉，不是“让模型更强”，而是“先让训练没有那么难”。

它到底做了什么

假设我们在某一层拿到一个 mini-batch 的输出。对每一个 feature channel，BatchNorm 会先在这个 batch 里计算均值和方差：

μB=1m∑i=1mxi,σB2=1m∑i=1m(xi−μB)2\mu_B = \frac{1}{m}\sum_{i=1}^m x_i,\qquad \sigma_B^2 = \frac{1}{m}\sum_{i=1}^m (x_i-\mu_B)^2 μB​=m1​i=1∑m​xi​,σB2​=m1​i=1∑m​(xi​−μB​)2

然后把每个样本做标准化：

x^i=xi−μBσB2+ϵ\hat{x}_i = \frac{x_i-\mu_B}{\sqrt{\sigma_B^2+\epsilon}} x^i​=σB2​+ϵ​xi​−μB​​

到这一步，我们把一组原本可能偏移、可能尺度混乱的数，拉回了均值接近 0、方差接近 1 的区间。这样做的结果，是后面那一层不必每次都重新适应一个完全不同的数值世界。

但 BatchNorm 没有停在这里。它还会再加上一个可学习的缩放和平移：

yi=γx^i+βy_i = \gamma \hat{x}_i + \beta yi​=γx^i​+β

这一步很关键。如果只做标准化，模型表达能力可能会被硬性限制住。加入 \gamma 和 \beta 之后，网络依然可以自己决定：到底要不要保留某个尺度，要不要把分布往某个方向拉回去。也就是说，BatchNorm 不是粗暴地把数值压扁，而是先整理，再把主导权还给模型。

这也是它最干净的一点：它提供秩序，但不直接替模型做决定。

为什么它常常有效

BatchNorm 在实现上不复杂，但它带来的好处很实际。

第一，它通常允许你用更大的学习率。因为每层收到的输入尺度比较稳，优化器不用那么小心翼翼地避开数值爆炸的边界。

第二，它会让梯度传播更平稳。这里不是说梯度从此不会消失或爆炸，而是说，很多原本会因 activation 饱和、数值尺度失衡而被放大的问题，会被削弱一些。尤其在 sigmoid 或 tanh 这类容易饱和的非线性附近，这件事格外重要。

第三，它常常带一点正则化效果。因为训练时用的是 batch 内的均值和方差，而每个 mini-batch 都带一点抽样噪声，这等于在训练过程中引入了轻微扰动。这种不完全确定性，有时反而能让模型不那么容易死死贴住训练集。

如果要把这些效果收成一句话，我会说：BatchNorm 让优化地形变得比较平顺。它不一定改变问题本身，但它让你不必总是在崎岖的边坡上落脚。

训练和推理，其实是两个时刻

很多人第一次实现 BatchNorm，容易忽略一件小事：训练时和推理时，它用的统计量并不一样。

训练时，BatchNorm 使用当前 mini-batch 的均值与方差，因为这些量是眼前就能算出来的。可是在推理时，我们往往一次只送进一条数据，或者 batch size 很不稳定，这时候再用当前 batch 的统计量就不合适了。

所以在训练过程中，BatchNorm 通常还会维护一组 running mean 和 running variance。到了 inference 阶段，就改用这组移动平均的统计量。

这个细节表面上只是工程问题，其实也提醒了一件事：BatchNorm 的稳定，并不是某种静止的稳定。它是在训练的流动里估计一个比较可靠的中心，然后在推理时把这个中心保存下来。

它不是万能药

如果把 BatchNorm 说成深度学习里的万灵丹，那就过头了。

它依赖 batch 统计量，所以 batch size 太小的时候，估计会变得很吵；在 RNN 这类时间步关联很强、或是微小 batch 训练的场景里，它也未必是最自然的选择。这也是为什么后来又有了 LayerNorm、InstanceNorm、GroupNorm 这些不同方向的 normalization 方法。

而且，现在大家也知道，BatchNorm 的成功不必被绑死在最初那个 internal covariate shift 的叙事上。更被广泛接受的理解是：它改善了优化问题的条件，让 loss landscape 没那么尖锐，让参数更新对局部扰动没那么脆弱。换句话说，它未必是消除了漂移本身，而是让模型在漂移存在时，依然比较走得动。

这个差别不只是措辞上的修补。它让我们明白，一个方法之所以有效，和它最初被介绍时的故事，未必永远是同一件事。

原因很直接。线性变换之后，先把数值分布整理好，再送进非线性函数，通常会比较稳。对卷积网络来说，BatchNorm 会对每个 channel 做 normalization，并在 batch 与空间维度上统计均值和方差；对全连接层，则是对每个 feature 维度处理。

当然，架构不同，细节也可能不同。但如果只是建立第一个正确直觉，先记住这个位置通常就够用了。

最后再看它一眼

我一直觉得，BatchNorm 的气质很像一种节制。

它不替你设计模型，不替你选数据，不替你保证收敛。它只是在每一层之间，做一个很短的停顿：把刚刚那批有些偏、有些乱、有些太大或太小的数，轻轻拉回一个可工作的范围。然后训练才得以继续往前。

深度学习里有很多方法都在追求更强、更快、更大。BatchNorm 没有那种姿态。它更像是在说，先把地面铺平一点。很多时候，事情之所以能走远，不是因为一开始跑得多猛，而是因为每一步都还踩得到实处。
