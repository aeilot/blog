CSAPP Bomb Lab 解析

做完了 CSAPP Bomb Lab，写一篇解析。

题目要求

运行一个二进制文件 bomb，它包括六个"阶段(phase)“，每个阶段要求学生通过 stdin 输入一个特定的字符串。如果输入了预期的字符串，那么该阶段被"拆除”，进入下一个阶段，直到所有炸弹被成功"拆除"。否则，炸弹就会"爆炸"，打印出"BOOM!!!"

环境

这个系统是在 x86_64 Linux 上运行的，而笔者的环境是 ARM 架构的 macOS (Apple Silicon)。

捣鼓了半天 docker，虚拟化一个 x86_64 Ubuntu 出来，结果里面的 gdb 不能用，不想折腾。

发现 educoder 上面有环境，可以直接用，而且免费，于是就在 educoder 上面完成了本实验。

地址：https://www.educoder.net/paths/6g398fky

前置知识

本实验要求掌握 gdb 的一些指令。

1. 启动与退出 (Startup & Exit)

指令 缩写 描述
gdb executable - 启动 GDB 并加载可执行文件。
run [args] r 开始运行程序。如果有命令行参数，跟在后面（如 r input.txt）。
quit q 退出 GDB。
start - 运行程序并在 main 函数的第一行自动暂停（省去手动打断点的麻烦）。
set args ... - 设置运行时的参数（在 r 之前使用）。

2. 断点管理 (Breakpoints)

指令 缩写 描述 示例
break <loc> b 设置断点。支持函数名、行号、文件名:行号。 b main
b 15
b file.c:20
info breakpoints i b 查看当前所有断点及其编号 (Num)。 -
delete <Num> d 删除指定编号的断点。不加编号则删除所有。 d 1
disable/enable <Num> - 暂时禁用或启用某个断点（保留配置但不生效）。 disable 2
break ... if <cond> - 条件断点：仅当条件为真时才暂停（非常有用）。 b 10 if i==5

3. 执行控制 (Execution Control)

指令 缩写 描述 区别点
next n 单步跳过。执行下一行代码。 如果遇到函数调用，不进入函数内部，直接执行完该函数。
step s 单步进入。执行下一行代码。 如果遇到函数调用，进入函数内部逐行调试。
continue c 继续运行，直到遇到下一个断点或程序结束。 -
finish - 执行直到当前函数返回。 当你不小心 s 进了一个不想看的库函数时，用这个跳出来。
until <line> u 运行直到指定行号。 常用于快速跳出循环。

4. 查看数据 (Inspection)

指令 缩写 描述
print <var> p 打印变量的值。支持表达式（如 p index + 1）。
display <var> - 持续显示。每次程序暂停时，自动打印该变量的值（适合跟踪循环中的变量）。
info locals - 打印当前栈帧中所有局部变量的值。
whatis <var> - 查看变量的数据类型。
ptype <struct> - 查看结构体或类的具体定义（成员列表）。
x /nfu <addr> x 查看内存。n是数量，f是格式(x=hex, d=dec, s=str)，u是单位(b=byte, w=word)。
例如：x/10xw &array (以16进制显示数组前10个word)。

5. 堆栈与上下文 (Stack & Context)

指令 缩写 描述
backtrace bt 查看调用栈。显示程序崩溃时的函数调用路径（从 main 到当前函数）。
frame <Num> f 切换到指定的堆栈帧（配合 bt 看到的编号）。切换后可以用 p 查看该层函数的局部变量。
list l 显示当前行附近的源代码。

6. 提升体验：TUI 模式 (Text User Interface)

  • layout src：屏幕分为两半，上面显示源代码和当前执行行，下面是命令窗口。（强烈推荐）
  • layout asm：显示汇编代码。
  • layout split：同时显示源代码和汇编。

这两个整数依次存为 rsp+8, rsp+c

这里判断了第一个数，如果这个数大于 7，就会引爆

我们把第一个整数存入 eax，这里很明显是一个 switch 的跳转表：0x402470 + 8*rax

eax 和 rax 实际上是同一个东西，前者是这个寄存器的前 32 位，后者是这个寄存器的完整 64 位，这是历史遗留产物，实际上，还有 ax, ah, al，为了向后兼容而保留。

我们来读取 10 个，x/10x 0x402470，得到：

我们随便选一个就能得到正确答案，如，0 对应 0x00400f7c

第二个数和 eax 比较，相等就拆除成功

我们得到第二个数 0xcf = 207

于是，答案是 0 207

实际上，答案并不唯一，观察代码可以知道，每一个 switch 分支中，都对应了一个第二个整数的正确答案。

偏置

整数除法要求向零舍入。对于正数，向下舍入；对于负数，向上舍入。除以2的幂可以用右移操作替代。

但是，对于补码右移，很可能出现舍入错误。

我们进行右移的时候，其实是舍去了最低位，是一种向下取整
