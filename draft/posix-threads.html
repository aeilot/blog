Write a Reader Writer Lock with pthreads

Pthreads or POSIX Threads (coming in <pthreads.h>) allows a program to control multiple different flows of work that overlap in time. In this tutorial, we’ll cover some of the basic usages of <pthreads.h>, followed by an example — the Reader-Writer Lock.

Threads

A thread is a single sequence stream within a process.

Threads are like lightweight processes. They allow multiple tasks to run simultaneously, improving program efficiency.

Threads share with other threads their code section, data section and OS resources, while having their own program counters (PC), register sets, and runtime stack spaces.

In C, we use the POSIX Threads (or pthreads) to implement multithreading, defined in the <pthread.h> header file.

What’s <pthreads.h> all about?

pthreads comes with over 100 thread procedures in <pthreads.h>, all prefixed pthread_. They can be categorized into five groups:

  1. Basic Thread Management
  2. Mutexes
  3. Condition variables
  4. Thread synchronization
  5. Spinlocks

pthreads Data Structures

The POSIX threads API (usually provided by the pthread library on Unix/Linux systems) defines several core data structures used to represent threads and synchronization primitives.

  1. pthread_t: represents a thread identifier.
  2. pthread_attr_t: represents thread attributes used during thread creation, such as the stack size, stack address, scheduling policy etc.
  3. pthread_mutex_t: represents a mutex, or mutual exclusion lock.
  4. pthread_mutexattr_t: attributes for a mutex, used to configure mutex behavior.
  5. pthread_cond_t: represents a condition variable.
  6. pthread_condattr_t: condition variable attributes.
  7. pthread_rwlock_t: A reader-writer lock.
  8. pthread_barrier_t: A barrier synchronization primitive.
  9. pthread_key_t: Used for thread-local storage (TLS).

2. Thread Lifecycle

After pthread_create, a thread has two possible lifecycle management modes: joinable or detached.

  • pthread_join is used when another thread (usually the parent) must wait for the thread to finish and collect its return value.
  • pthread_detach marks a thread so that its resources are automatically released when it exits.
